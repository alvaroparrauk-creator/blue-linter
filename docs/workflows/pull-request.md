# Pull Request Workflow

## Before Opening A PR

- Confirm the branch is scoped to one task.
- Run relevant verification checks.
- Run Docker verification when required by the project.
- Update docs and backlog when needed.
- Confirm no unrelated local changes are included.

## PR Description

Include:

- summary of changes
- tests and verification run
- Docker status when relevant
- docs or backlog updates
- screenshots or GUI notes when useful
- known limitations or deferred follow-up

## After Opening A PR

Send the user a validation note with:

- what changed
- how to test the feature
- useful non-GUI checks
- verification commands run
- PR link
